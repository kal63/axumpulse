# AxumPulse Frontend Changelog

All notable changes to the AxumPulse frontend application will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.0.0] - 2026-03-10

### 🎨 **UI/UX Enhancements**

#### Added
- **Hero Section Improvements**:
  - Hero-style headers for daily challenges, games & leaderboard pages
  - Dynamic language selector dropdown in header
  - Enhanced mobile BottomNavigation with internationalization support
  - Updated subscription pricing display (4 ETB minimum)

- **Trainer Portfolio System**:

  - Trainer profile pages with customizable layouts
  - Specialties display with proper array validation
  - Bio and contact information sections

- **Medical System Frontend**:
  - Medical application status page with real-time updates
  - Enhanced MedicalApplyPage with language fetching and validation

  - Doctor consultation booking interface

- **Subscription Management**:
  - Subscription checks and indicators across all user pages

- **Gaming Features**:

## Versioning Guidelines

### Frontend Versioning Strategy
- **MAJOR**: Breaking UI changes, complete redesigns, API contract changes
- **MINOR**: New features, significant UI improvements, new components
- **PATCH**: Bug fixes, small UI tweaks, performance improvements

### Development Notes
- All frontend changes are tracked in the `axumpulse` directory
- Components follow React best practices with TypeScript
- UI components use Tailwind CSS for styling
- State management through React Context and hooks
